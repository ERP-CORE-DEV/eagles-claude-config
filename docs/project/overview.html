EAGLES Pro

A universal configuration framework that turns Claude Code into a fully equipped development environment with agents, skills, hooks, rules, MCP servers, and a DSL engine.

💡

What this project does

EAGLES (Enhanced Agents, Guidelines, Libraries, Extensions, and Skills) solves a fundamental problem: every time you start a new Claude Code session, you lose context -- your coding patterns, architecture decisions, security rules, and team conventions all reset to zero. EAGLES eliminates this by providing a persistent, extensible framework that lives in your ~/.claude/ directory and loads automatically on every session.

The framework combines five complementary systems:

  • 1.24 specialized agents that handle complex workflows (TDD, code review, security scanning, architecture design, documentation generation)
  • 2.72 slash-command skills that execute specific development tasks (/tdd, /code-review, /scaffold-microservice, /gitleaks)
  • 3.10 MCP servers that connect Claude Code to external tools (GitHub, Chrome DevTools, filesystem, team-sync)
  • 4.7 hooks across 5 lifecycle events that enforce quality gates automatically (secret detection, coverage reminders, doc-blocker)
  • 5.A TypeScript DSL engine that parses shortcut commands like @arch:ms @deploy:k8s --scale=3 into structured execution plans

EAGLES is universal. It works with .NET, Node.js, Python, Java, Go, and any other technology stack.

👥

Who it is for

Individual developers

Using Claude Code CLI who want persistent coding standards, automated quality gates, and reusable workflows across all projects

Development teams

Sharing a common Claude Code configuration through a Git repository (install once with setup_claude.ps1, everyone gets the same setup)

Tech leads & architects

Who need to enforce patterns (Controller-Service-Repository, GDPR, secret scanning) across a team without manual review

Design philosophy

1. Universal, not stack-specific

EAGLES works with any language, framework, or deployment target. Stack-specific rules live in dedicated subdirectories and load contextually.

2. Persistent across sessions

Everything lives on disk in ~/.claude/. Claude Code loads it automatically. No setup commands needed after installation.

3. Composable

You combine agents, skills, hooks, and rules independently. Use /tdd without the orchestrator. Add a hook without touching any agent.

4. Context-rot resistant

The orchestrator delegates to sub-agents via the Task tool, giving each phase a fresh 200K-token context. The PostToolUse hook suggests /compact every 25 calls.

5. Team-distributable

A single Git repo holds the full configuration. setup_claude.ps1 copies everything to ~/.claude/ on each developer's machine.
