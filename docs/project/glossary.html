Glossary

Definitions of all EAGLES-specific terms and the external frameworks it references.

🦅

EAGLES terms

Agent

A Markdown file in ~/.claude/agents/ that defines specialized behavior for Claude Code. Each agent has a name, description, list of allowed tools, model preference (opus or sonnet), and a mode (primary, subagent, or secondary). EAGLES ships with 24 agents.

Skill

A slash command that you invoke in Claude Code (e.g., /tdd, /code-review, /gitleaks). Each skill is a directory under ~/.claude/skills/ containing a SKILL.md file with YAML frontmatter and Markdown instructions. EAGLES ships with 72 executable skills.

Hook

A command that runs automatically at a specific lifecycle point in Claude Code. Hooks are defined in settings.json under the hooks key. They receive tool input/output as JSON on stdin and can either allow (exit 0) or block (exit 2) tool calls. EAGLES uses 7 hooks across 5 lifecycle events.

Rule

A plain Markdown file in ~/.claude/rules/ that Claude Code loads as a system instruction. Rules enforce coding standards, architecture patterns, and team conventions without requiring the developer to remember them. EAGLES ships with 10 rule files.

MCP server

A Model Context Protocol server that extends Claude Code with external tool capabilities. MCP servers are configured in .mcp.json and communicate via stdio. Examples: GitHub API, Chrome DevTools, filesystem access, team-sync. EAGLES configures 10 MCP servers.

DSL engine

A TypeScript application that parses shortcut commands like @arch:ms @deploy:k8s --scale=3 into structured execution plans. The engine has three stages: Parse (tokenizer), Validate (category and skill checking), Generate (execution plan output). It supports 17 prefix categories.

Context rot

The gradual degradation of Claude Code response quality as the context window fills up during a long session. After approximately 50+ tool calls, responses may become less accurate. EAGLES mitigates this through the orchestrator agent (sub-agent delegation with fresh contexts), the context-rot counter hook, and the /compact command.

GSD (Get Stuff Done)

An execution framework used by the EAGLES orchestrator agent. GSD decomposes complex tasks into phased waves, where each wave runs in parallel sub-agents with fresh 200K-token contexts. The workflow follows: PLAN (decompose task), WAVE execution (parallel sub-agents), STATE tracking, VERIFY (build + test).

Two-tier skill system

EAGLES uses two layers of skills: (1) YAML reference skills (103 patterns covering the full development lifecycle), and (2) Markdown executable skills (72 slash commands). The executable layer implements or references the reference layer.

Primary / Subagent / Secondary

Primary: runs as the main agent, interacts with the user. Subagent: spawned by another agent via Task tool, runs in fresh context. Secondary: runs alongside a primary agent for parallel tasks.

Doc-blocker

A PreToolUse hook (~/.claude/hooks/doc-blocker.py) that intercepts all Write tool calls for .md files. It blocks writes unless the file path contains an allowed string (/docs/, CLAUDE, README, CHANGELOG, SKILL, rules/, agents/, memory/).

FRAMEWORK-MANIFEST.yaml

The master index file that describes the entire reference configuration library: architecture patterns, workflows, stacks, deployment strategies, testing frameworks, integration contracts, debugging tools, 103 skills, and 2 agents. Version 1.2.0.

External frameworks

Diataxis

A documentation framework by Daniele Procida that organizes content into four types: Tutorials (learning-oriented), How-to guides (task-oriented), Explanation (understanding-oriented), and Reference (information-oriented). EAGLES uses Diataxis to structure its docs/project/ documentation.

Arc42

A template for software architecture documentation created by Gernot Starke and Peter Hruschka. It has 12 sections (S1-S12). EAGLES uses Arc42 sections to structure ARCHITECTURE.md.

C4 Model

A set of hierarchical abstractions for software architecture diagrams by Simon Brown. Four levels: L1 System Context, L2 Container, L3 Component, L4 Code. EAGLES references C4 levels in architecture documentation and diagram generation.

MCP (Model Context Protocol)

An open protocol by Anthropic for connecting AI models to external tools and data sources. MCP servers communicate via stdio and expose tools that Claude Code can call.

Google Developer Documentation Style Guide

A writing style guide that emphasizes second-person voice, active voice, present tense, conditions-first sentence structure, and no filler words. EAGLES documentation follows this guide.

12-Factor App

A methodology for building SaaS applications. EAGLES references Factor III (Store config in the environment) for its configuration documentation.

Architecture terms

Controller-Service-Repository

The architecture pattern enforced by EAGLES rules for .NET projects. Controllers handle HTTP concerns, services handle business logic and validation, repositories handle data access. Explicitly chosen over CQRS/MediatR for simplicity.

CosmosDB SDK 3.54 direct

The data access approach enforced by EAGLES rules. Uses the Azure Cosmos DB .NET SDK v3 directly (not Entity Framework Core) for full control over partition keys, request units (RU), and query optimization.

GDPR/CNIL compliance

European data protection requirements enforced by EAGLES security rules. Includes: no PII in logs, AnonymizeXxx() methods on personal data entities, IsAnonymized flag checks before data export.

French HR domain

The primary business domain for the RH-OptimERP project. Includes: SMIC (minimum wage), CDI/CDD/CDIC (contract types), CPF (personal training account), OPCO (skills operators), RNCP/RS (certification codes), and Code du Travail (French labor law).
