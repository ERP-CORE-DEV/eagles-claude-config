permissions.allow 177 entries

Glob-style patterns that whitelist specific tool calls.

JSON 
"allow": [
  "Bash(git:*)",        // All git commands
  "Bash(dotnet:*)",     // All dotnet commands
  "Bash(npm:*)",        // All npm commands
  "Read(//c/**)",       // Read any file on C: drive
  "Write(//c/**)",      // Write any file on C: drive
  "mcp__github__*"      // All GitHub MCP tools
  // ... 171 additional patterns
]

permissions.deny 6 entries

Patterns that are always blocked regardless of allow list. Deny rules take precedence.

JSON 
"deny": [
  "Bash(rm -rf:*)",           // Recursive force delete
  "Bash(rmdir:*)",            // Directory removal
  "Bash(del /s:*)",           // Windows recursive delete
  "Bash(git reset --hard:*)", // Hard git reset
  "Bash(git push --force:*)", // Force push
  "Bash(git clean -f:*)"     // Force clean
]

Environment variables

Set these in your host shell before launching Claude Code. The ${VAR} syntax in .mcp.json reads from the system environment at runtime.

VariableRequired byDescription
GITHUB_PERSONAL_ACCESS_TOKENgithub MCPPersonal access token with repo and read:org scopes
QONTO_CLIENT_IDqonto MCPOAuth 2.0 client ID for Qonto API
QONTO_CLIENT_SECRETqonto MCPOAuth 2.0 client secret for Qonto API

Security note

Never hardcode these values in .mcp.json or any tracked file. Set them in your shell profile (~/.bashrc, ~/.zshrc, or Windows environment variables) so they are available without appearing in version control.
