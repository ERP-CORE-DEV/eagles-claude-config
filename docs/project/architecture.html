S4 -- Solution Strategy

File formats

FormatUsed forWhy
Markdown + YAMLAgents and SkillsClaude Code natively loads .md files as context
Plain MarkdownRulesClaude Code loads rules/ as system instructions
JSONSettings and MCPClaude Code expects JSON for config files
TypeScriptDSL engineType safety for parsing shortcut commands
PythonHooksCross-platform compatibility
YAMLReference configsMore readable for nested configurations

Key strategic decisions

Slash commands over DSL for daily use

Developers type /tdd, /code-review. The DSL engine is for composing complex multi-config setups.

Agents as Markdown, not code

Agent behavior is defined entirely in Markdown. Edit an agent by editing a text file.

Hooks as Python one-liners

Every hook is a python3 -c command. Avoids shell compatibility issues across Windows/macOS/Linux.

Two-tier skill system

YAML skills are reference definitions (103). Markdown skills are executable slash commands (72).

S6 -- Runtime Behavior

Skill invocation flow

  1. 1.Claude Code reads ~/.claude/skills/tdd/SKILL.md
  2. 2.Frontmatter specifies agent: tdd-guide
  3. 3.Agent loaded from ~/.claude/agents/tdd-guide.md
  4. 4.Agent instructions activate (RED-GREEN-REFACTOR)
  5. 5.$ARGUMENTS replaced with user input
  6. 6.Agent uses tools (Read, Write, Edit, Bash)
  7. 7.PostToolUse hooks fire after each tool call

Orchestrator wave flow

  1. 1.Orchestrator loads from agents/orchestrator.md
  2. 2.PLAN: Creates .planning/ directory
  3. 3.WAVE 1: Spawns sub-agents (parallel, fresh contexts)
  4. 4.Updates STATE.md, checks verification gates
  5. 5.WAVE 2: Implementation sub-agents
  6. 6.Updates STATE.md
  7. 7.WAVE 3: Integration testing
  8. 8.VERIFY: Build + test + coverage check

Hook execution flow

  • -PreToolUse: exit 0 = allow, exit 2 = block with message
  • -PostToolUse: stdout shown as warning (never blocks)
  • -Stop: session persistence
  • -SessionStart: context recovery
  • -PreCompact: saves WIP before trim
  • -JSON on stdin: {tool_input, tool_result}

S8 -- Crosscutting Concerns

Windows compatibility

  • -All hooks use python3 -c instead of shell scripts
  • -Explicit os.path.expanduser('~') (tilde does not expand on Windows)
  • -Backslashes normalized to forward slashes before path matching

Context rot prevention

  • 1.Orchestrator sub-agents: fresh 200K-token contexts via Task tool
  • 2.Context-rot counter: suggests /compact every 25 calls
  • 3.PreCompact hook: saves critical context before compaction

Documentation guard

  • -doc-blocker.py intercepts all Write calls for .md files
  • -Allowed paths: /docs/, CLAUDE, README, CHANGELOG, SKILL, rules/, agents/, memory/
  • -Blocks with exit code 2 and descriptive error message

Secret protection

  • 1.PreToolUse hook: blocks git commit without prior gitleaks scan
  • 2..gitleaks.toml: custom rules for Azure keys, CosmosDB strings
